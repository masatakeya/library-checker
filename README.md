# 図書館蔵書チェッカー (Library Collection Checker)

本アプリは、指定した地域の図書館にある本の貸出状況（蔵書状況）を簡単に検索・確認できるシングルページWebアプリケーションです。
外部API（Google Books API, openBD API, カーリルAPI）を利用しており、バックエンドサーバ不要でフロントエンド（HTML/CSS/JavaScript）のみで動作します。

## 主な機能と特徴

1. **本の検索**
   - **タイトル・著者名**: Google Books APIを利用して本を検索します。
   - **ISBN**: openBD APIを利用して、ISBN（10桁または13桁）から本を正確に特定します。
2. **図書館の検索とフィルタリング**
   - 指定した「都道府県＋市区町村」（例: 東京都 府中市）から、その地域の図書館システムを検索します。
   - **大学・専門機関の除外機能**: 検索時に貸出対象外となりやすい大学や専門機関をデフォルトで除外し、公共図書館のみを瞬時にリストアップします（チェックボックスで含めることも可能）。
   - 直近の検索履歴はブラウザ（`localStorage`）に最大3件まで保存され、ワンタップで再検索が可能です。
3. **高精度な蔵書状況の確認**
   - カーリルAPIを利用し、選択した本が目的の図書館にあるかどうかをリアルタイムでポーリング検索します。
   - **13桁/10桁ISBNの自動並行検索**: 13桁のISBNで検索した場合でも、内部で自動的に10桁のISBNを算出し、同時にAPIへ照会します。これにより、旧来のシステム（古いOPAC等）を使用している図書館でのヒット率が大幅に向上しています。

## 利用技術

- HTML5 / CSS3 (プレーンな構成)
- Vanilla JavaScript (ES6+, Fetch APIによるCORS通信)
- **利用外部API**:
  - [Google Books APIs](https://developers.google.com/books)
  - [openBD API](https://openbd.jp/)
  - [カーリル図書館API](https://calil.jp/doc/api_ref.html) (CORS JSON方式で通信)

## 使い方（セットアップ）

本アプリはサーバーレスで動作するため、GitHub Pages等の静的ホスティングでそのまま公開可能です。

1. ファイル一式をサーバーに配置するか、ローカル環境に保存します。
2. **APIキーの設定**: プロジェクトルートにある `config.js` をテキストエディタで開き、`CALIL_API_KEY: ""` の中にご自身のカーリルAPIキーを入力して保存してください。
   - ※ APIキーをお持ちでない場合は、[カーリルAPIダッシュボード](https://calil.jp/api/dashboard/)から無料で取得可能です。
   - `config.js` にキーを設定しておくことで、ページにアクセスするユーザー全員が個別設定なしで利用できるようになります。
3. `index.html` をブラウザで開きます。
4. 画面の指示に従い「1. 本を探す」→「2. 図書館を選ぶ」の順に進めると、「3. 蔵書状況」にリアルタイムな貸出状況が表示されます。
