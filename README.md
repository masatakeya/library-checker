# 図書館蔵書チェッカー (Library Collection Checker)

本アプリは、指定した地域の図書館にある本の貸出状況（蔵書状況）を簡単に検索・確認できるシングルページWebアプリケーションです。
外部API（Google Books API, openBD API, カーリルAPI）を利用しており、バックエンドサーバ不要でフロントエンド（HTML/CSS/JavaScript）のみで動作します。

## 機能

1. **本の検索**
   - **タイトル・著者名**: Google Books APIを利用して本を検索します。
   - **ISBN**: openBD APIを利用して、ISBN（10桁または13桁）から本を特定します。
2. **図書館の検索**
   - 指定した「都道府県＋市区町村」（例: 東京都 調布市）から、その地域の図書館システムを検索します。
   - 直近の検索履歴はブラウザ（`localStorage`）に最大3件まで保存され、ワンタップで再検索が可能です。
3. **蔵書状況の確認**
   - カーリル共有API（JSONPによるポーリング）を利用し、選択した本が目的の図書館で「貸出中」か「蔵書あり」かを一覧表示します。
   - 予約ページが存在する場合は、図書館システムの詳細ページへのリンクを表示します。

## 利用技術

- HTML5 / CSS3 (プレーンな構成)
- Vanilla JavaScript (ES6+, 非同期処理, JSONP)
- **外部API**:
  - [Google Books APIs](https://developers.google.com/books)
  - [openBD API](https://openbd.jp/)
  - [カーリル図書館API](https://calil.jp/doc/api_ref.html)

## 使い方

1. `index.html` をブラウザで開きます（サーバーは不要、ローカル環境のダブルクリックでも動作します）。
2. アプリ右上の **「⚙ APIキー設定」** ボタンを開き、カーリルAPIキーを設定・保存します。
   - ※ APIキーをお持ちでない場合は、[カーリルAPIダッシュボード](https://calil.jp/api/dashboard/)から無料で取得可能です。
   - 入力したキーはブラウザ内部の `localStorage` にのみ保存され、安全に扱われます。
3. **「1. 本を探す」** で探したい本を検索し、リストから対象の本を選択します。
4. **「2. 図書館を選ぶ」** で地域を入力して検索します。
5. **「3. 蔵書状況」** に結果がロードされ表示されます。

## 特別な仕様・工夫

- **JSONP通信**: カーリルAPIとの通信はCORSの制約を回避するためJSONP（動的 `<script>` タグの挿入）を利用しています。これにより、ローカルの `file://` プロトコルから開いてもAPI通信が成立するよう設計されています。
- **モダンなUIUX**: ローディングアニメーション（スピナー）や、直感的なステッパー（1→2→3）、レスポンシブなCSS設定により、モバイルデバイスでもPCでも快適に利用可能です。
